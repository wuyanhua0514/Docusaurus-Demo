# RM Introduction

RM平台介绍
一.平台介绍
1. 什么是RM？
RM是Robot Management平台的简称，是赛舵智能（AI RUDDER）自研的web端机器人制作与管理平台。
基于RM，可以高效、低成本的搭建对话系统，根据业务场景自动发起智能机器人电话外呼或虚拟客服接听任务，通过人与机器人的语音对话交互收集业务结果，并对数据加以统计处理，获取用户反馈。
在RM，企业可以创建对话机器人、查看外呼明细、查看数据报表，根据数据情况及时调整机器人对话策略，以最小成本创造最大价值。目前赛舵智能公司业务已覆盖全球数十个国家，为东南亚、印度、非洲和美洲的 200 多家客户，提供企业级服务并获得最佳实践，并在金融、保险、电商、物流等10多个垂直行业形成大量客户成功案例和智能交互解决方案。

三.功能详述
1. 首页
首页是登录进入RM之后的欢迎页面，能够登录RM的用户都有权限访问首页。

2. 机器人中心
  机器人中心是用来制作、测试、管理对话机器人的核心部分。
2.1 机器人管理
概述
  机器人管理模块主要对账户名下的所有机器人进行维护管理，查询当前机器人的在线状态等；通过版本管理可以实现对机器人的历史版本进行维护管理，可以通过【复制】操作实现机器人版本的快速创建，同时可以【上线】【下线】等操作在线生效的版本；
[图片]
2.1.1 如何创建一个机器人
  登录RM平台后，您可以按照以下步骤构建您的第一个机器人。
  设计：在您正式开始构建机器人之前，这是需要准备的必要步骤；
  构建：在此阶段，需要进行必要的配置，您的机器人将在RM平台上构建成型；
  1. 设计/定义机器人
  建议您在设计机器人阶段可以遵循以下步骤：
    - 了解需求：为了设定机器人的服务范围，需要结合实际业务场景评估内外部需求；例如需要设计一款营销场景机器人对某款产品进行销售；
    - 设定目标：明确需求后需要将上述确定的需求范围转换为实际用例，以设定具体场景的实现目标；例如向用户介绍产品详情并收集用户意向；
    - 设计对话：定义机器人在于用户交互中每个可能场景的行为，以构建对话流程；
  明确定义了机器人的功能及理想场景，您就可以开始在RM上通过定义变量、意图及实体，设计话术并开始构建对话流程
  2. 构建机器人
  在RM平台上构建一个机器人通常包括以下步骤：
    - 选择需要构建的机器人基础信息
    - 构建机器人流程
    - 训练机器人
    - 测试机器人
    - 上线发布机器人
    - 优化改进
    1. 配置机器人基础信息
    登录RM后，进入机器人中心的机器人管理模块，点击【新建机器人】后，即可新建机器人；
    - 机器人的基础信息，包括机器人类型、国家、语言、音色等；
    - 配置机器人的引擎，包括ASR、TTS、NLU等模型；
[图片]
    2. 构建机器人流程
    完成机器人的基础信息创建后，您可继续为机器人配置问答能力，该部分为机器人的核心对话能力的配置，您可以使用以下方法为您的机器人定义对话流程：
    - 知识问答：为您的机器人配置知识库，实现知识问答能力；
    - 任务流程：为您的机器人配置任务流程，实现多轮对话的问答能力；
[图片]
    平台为您提供可视化的机器人任务流程工作，您可通过【可视化流程配置】为机器人创建对话流程，通过画布【Action】及【Condition】的组件实现机器人的对话流程的设计；
    - Action节点：定义机器人的执行动作，包括回复话术、挂机、转人工、节点跳转等，以及为该对话节点配置问答策略配置，包括问答策略、填槽策略、外部服务调用等；
    - Condition节点：定义机器人后续流程的分支，通过内置的函数的组合设定分支条件，实现机器人流程的跳转；
[图片]
    3. 训练机器人
    在完成机器人问答能力及对话流程的创建后，您可以进行机器人的训练，使机器人能够更好的理解用户的意图，完成训练后，可以通过进一步的内部测试及训练数据来优化模型
    详细信息，请参阅5.2 意图训练及5.3 模型测试章节；
    4. 测试机器人
    构建并训练您的机器人后，需要通过进一步的测试来验证机器人实际表现，您可以通过文本对话测试、节点测试、电话拨测等方法来测试机器人的对话效果；
    详细信息，请参阅2.2 机器人测试章节；
    5. 上线发布
    机器人进行充分测试后，您就可以进行上线发布，完成机器人的制作流程；
    6. 优化建议
    上线机器人后，后续的跟踪优化也尤为重要。分析模型的预测效果，统计机器人每个节点的意图触发和消耗频次，以及对应节点的挂机频率可以帮助您更好的发现机器人对话流程的改进方向。
    详细信息，请参与6.1 意图工厂章节；
2.1.2 进阶能力
  1. CheckPoint流程处理
    CheckPoint流程处理主要帮助您处理对话过程中的可能随时会出现CheckPoint流程，例如用户抱怨、寻求帮助、没有回应、用户繁忙、无法识别等非主任务流程的对话处理；您可以通过触发等级的设定来控制意图处理的优先级；
  2. 填槽能力应用
    填槽是指通过用户回复的内容提取关键信息，用于补全用户意图，这些关键信息称之为"实体"；
    RM为您提供了系统内置的实体抽取能力，包括时间、数字、货币金额、姓名、地名等通用的实体识别能力，您可通过定义对话任务中的槽位信息，实现人机对话过程中的关键信息提取；
[图片]
  3. 变量应用
    变量是指对话过程中获取的固定信息或用户数据，主要用于对话流程的控制或话术的合成；
    RM为您提供的系统全局变量，包括来电号码、当前UTC时间、当前语言等变量以便您直接使用；也可通过自定义变量的方式，在对话任务开始前根据自定义变量的内容进行数据上传，实现对话过程中变量的应用；
[图片]
  4. 拟人化处理  @章微微 
    在语音对话机器人场景，为了提高机器人对话的拟人效果，RM为您提供了一系列的拟人化配置：
    - 等待用户回复时长设置
    - 允许用户打断时长控制
    - 打断检测敏感度设置
    - 语义打断设置
    - 打断次数限制
    - ....
  5. 高阶策略配置
    除以上对话能力配置以外，RM还为您提供了人机结合策略、语音信箱检测及留言策略等高阶应用，以适配您的不同业务场景需求
2.2 标签管理
  标签管理用来定义及配置每次对话之后产出的最终结果，分为大标签和事件标签两种，其中大标签是本轮对话的最终结论，事件标签是本轮对话的一些细分标志。
[图片]
  1. 大标签配置
  大标签配置是用来自定义大标签的展示内容（不影响机器人逻辑，只影响展示），目前支持中文、英文和印尼语的展示。
[图片]
  2. 大标签规则
  大标签规则可以设定特定的条件，对每一次机器人对话产出的大标签进行变更，是机器人逻辑的高级应用，当机器人流程中设定标签比较困难时，可以使用大标签规则设定。
[图片]
    大标签只有一个标签结果产出，示例：
      1. 客户1--条件：用户全程无文本；结果：大标签输出K标；
      2. 客户2--条件：在打招呼环节或者用户介绍环节，用户主动挂机；结果：大标签输出E标；
      3. 客户3--条件：用户在问题3环节的意图是replyYes或者replySatisfied，且在该环节用户主动挂机；结果：大标签输出A标；
      4. 客户4--条件：用户回答过问题1，或者问题2的number槽填上；结果：大标签输出A标；
  1. 事件标签配置
    事件标签配置中包含了基础标签的定义和事件标签的定义，是所有标签的基础。
[图片]
  2. 事件标签规则
  事件标签规则的设定方法和大标签规则类似，区别在于事件标签规则可以对某一机器人的每次对话设定多个可能产出的标签。
[图片]
    事件标签可以是多个标签结果，示例：
      1. 客户1
        1. 条件1：客户在prolusion1节点挂机或者全程无回复或者整通对话意图不明确；结果1：事件标签输出Uncertainty；
        2. 条件2：大标签为D标且任意节点意图为replyDeny，或者大标签为D标且任意节点意图为replyNotOwn；结果2：事件标签输出Denial_or_notown；
        3. 条件3：任意节点意图为askForHelp或askForComplain；结果3：事件标签输出为askHelpOrComplain；
      2. 客户2
        1. 条件1：大标签为E标且任意节点意图为replyBusy或者任意节点意图为callLater；结果1：事件标签输出Busy；
        2. 条件2：大标签为D标且任意节点意图为replyDeny或replyNotOwn；结果2：事件标签输出Deny；
2.3 机器人测试
    该模块用于对制作完成的机器人进行测试，保障机器人的质量是否符合预期。
[图片]
    自动化测试针对某一机器人的具体版本进行测试，每一个机器人版本对应一个机器人测试项目。
    1. 节点检查
      节点检查用于检查每个节点的话术及后续处理是否正确，如果预期的话术或处理和制作的机器人不同，将会提示错误。
[图片]
[图片]
      1. 新建节点测试之后，会对每个节点进行检查，需要手动选择每个节点的话术名称，点击check会进行比对，并展示结果情况，全部节点通过视为节点测试通过。
      2. 全部节点完成测试或者主动点击产出报告，会结束本次节点测试并产出对应报告。
    2. 文本测试
    文本测试和流程测试功能类似，用于验证机器人整体流程和逻辑是否正确。可以直接通过文本对话的方式，更加直观的展示对话过程和执行逻辑。相对于流程测试，无需提前编辑好所有的节点及预期结果。
[图片]
    3. 其他
    除节点检查和文本测试外，RM还有配置检查、流程测试、变量测试等模块，如需使用可向客户经理申请开通。
      
3. 通话记录
  通话记录用于查看成功接听的通话信息，可以具体查看该通通话录音，具体每一轮对话信息以及对应的执行过程。
3.1 通话列表
  展示有权限看到通话记录，目前存储半年时间。增加callID、robotName、taskID等多种筛选条件，点击详情可以查看到该通通话相关的具体信息。
[图片]
3.2 通话详情
  通话详情页面可以查看具体的通话录音。
[图片]
  - 标注1可以通过输入callID，点击【查询】按钮，快速切换想要查看的通话详情，【下载打点数据】和【下载流程详情json数据】为高级功能，用于查看过程中机器人具体执行数据，适合有一定开发基础的人员查看；
  - 标注2、3对话过程拆分成一问一答的形式，并将机器人对话内容下方标注了机器人流程名称step和话术名称，用户回复内容下方标注了机器人识别的意图intention、槽值slot和语言language；点击对应机器人话术下方的【播放】按钮，可以让录音跳转到对应节点播放；点击机器人话术或用户回复话术，右侧标注6的部分是执行过程，会对应展示执行信息；
  - 标注4是具体的录音播放进度条，点击【▶️】按钮可以从头播放本通录音，拖拽进度条可以从任意时间播放；
  - 标注5是整通对话的综合内容，包括语言、国家、机器人名称、版本号、大标签、事件标签等等内容；
  - 标注6是机器人执行过程，展示具体的执行过程，包括ASR识别情况，NLU、FAQ等执行结果情况等；
  - 如果发现识别错误或不准确的情况，可以直接点击【NLU异常反馈】等按钮，反馈badcase；
4. 内容管理
4.1 知识库
  知识库是一个面向用户的知识问答管理系统，预先维护好公司的知识库，即可在用户提出问题时，根据知识库的内容做出相应的回答。
4.1.1 新建知识库
  进入知识库页面，点击【创建新的知识库】，填写知识库名称、公司名称和默认语言后，完成知识库的创建。一个公司只能拥有一个知识库。
[图片]
4.1.2 知识点管理
  每个知识库由多个知识点组成，每个知识点代表对一类问题的集合与回答。点击【新增知识点】可进行知识点的新增。
  - 标签：知识点的分类标签，每个知识点可设置5个标签；
  - 标准问：最能代表该知识点的问题，会使用文本相似算法进行匹配；
  - 相似问：与标准问的相似问题，会使用文本相似算法进行匹配；
  - 默认回答：在对话过程中，识别出本知识点时默认进行的回答，支持文本和音频形式；
  - 扩展答案：在对话过程中，识别出本知识点时的其他回答，支持文本和音频形式。
  可使用页面上的【导出】和【导入】进行知识点的批量管理。
[图片]
4.1.3 知识点标签管理
  知识点的标签管理用于知识点所用的标签分类管理。
  点击【新增标签】可创建新的标签，点击【重命名】可以重新命名标签名称，点击【删除】将会删除该标签（删除标签分类可能导致通过标签配置的知识点不生效）。
[图片]
4.1.4 知识库训练
  点击【训练】按钮，将整个知识库的内容进行模型训练，训练完成后模型将会自动更新上线。
[图片]
4.2 词表管理
  词表管理功能为ASR模型的增强服务，分为【ASR热词】和【ASR口水词】两个模块。
4.2.1 ASR热词
  若认为机器人对某些特定词的识别效果不佳时，可以使用ASR热词来增强识别。常见的用法有添加公司名称、品牌名称等非通用性词汇。
[图片]
4.2.2 ASR口水词
  ASR口水词是机器人拟人化的一种工具，配置口水词的机器人，可以在与用户对话的过程中，当用户在说话时进行口水词的迎合，使其体验更加人性化。
[图片]
4.3 音频库
  音频库主要用于对系统音频素材的统一维护，通过音频组合的创建可以实现机器人制作过程中的直接引用，无需重复维护；
[图片]
5. 模型管理
5.1 模型配置
  模型配置是设置机器人可使用的各类模型数据，包括NLU、NER、TTS等。
